## Description

Define Qt-friendly payload dataclasses for the Analysis context. These are emitted by SignalBridge signals to UI widgets.

## Acceptance Criteria

### Report Payloads
- [ ] #1 `ReportGeneratedPayload`: report_id (str), report_type (str), title (str), row_count (int), generated_at (str)
- [ ] #2 `ReportExportedPayload`: report_id (str), file_path (str), format (str)

### Query Payloads
- [ ] #3 `QueryExecutedPayload`: query_id (str), sql (str), row_count (int), column_names (list[str]), execution_ms (int)

### Visualization Payloads
- [ ] #4 `ChartGeneratedPayload`: chart_id (str), chart_type (str), title (str), data_points (int)
- [ ] #5 `GraphGeneratedPayload`: graph_id (str), node_count (int), edge_count (int)
- [ ] #6 `WordCloudGeneratedPayload`: cloud_id (str), word_count (int), top_words (list[str])

### Journal Payloads
- [ ] #7 `JournalEntryCreatedPayload`: entry_id (str), title (str), created_at (str)
- [ ] #8 `JournalEntryUpdatedPayload`: entry_id (str), title (str), updated_at (str)

### Design Rules
- [ ] #9 All payloads are frozen dataclasses
- [ ] #10 Use primitive types only
- [ ] #11 Include summary data for display (counts, titles)
