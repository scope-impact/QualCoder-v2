## Description

Define Qt-friendly payload dataclasses for the Agent context. These are emitted by SignalBridge signals to UI widgets.

## Acceptance Criteria

### Suggestion Payloads
- [ ] #1 `SuggestionGeneratedPayload`: suggestion_id (str), tool_name (str), description (str), confidence (float), preview (str)
- [ ] #2 `SuggestionApprovedPayload`: suggestion_id (str), tool_name (str), result_summary (str)
- [ ] #3 `SuggestionRejectedPayload`: suggestion_id (str), tool_name (str), reason (str?)

### Session Payloads
- [ ] #4 `SessionStartedPayload`: session_id (str), model_name (str), started_at (str)
- [ ] #5 `SessionEndedPayload`: session_id (str), duration_seconds (int), actions_count (int)

### Trust Payloads
- [ ] #6 `TrustLevelChangedPayload`: tool_name (str), old_level (str), new_level (str)

### Action Payloads
- [ ] #7 `AgentActionExecutedPayload`: action_id (str), tool_name (str), description (str), success (bool), timestamp (str)
- [ ] #8 `AgentMessageReceivedPayload`: message_id (str), role (str), content (str), timestamp (str)

### Statistics Payloads
- [ ] #9 `AgentStatsUpdatedPayload`: session_id (str), total_actions (int), approved (int), rejected (int), pending (int)

### Design Rules
- [ ] #10 All payloads are frozen dataclasses
- [ ] #11 Use primitive types only
- [ ] #12 Include human-readable descriptions for activity display
