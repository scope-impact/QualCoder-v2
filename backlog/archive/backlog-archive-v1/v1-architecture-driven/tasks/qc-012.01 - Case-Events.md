## Acceptance Criteria
<!-- AC:BEGIN -->
### Case Lifecycle Events (frozen dataclasses extending DomainEvent)
- [ ] #1 `CaseCreated` event: case_id, name, owner
- [ ] #2 `CaseRenamed` event: case_id, old_name, new_name
- [ ] #3 `CaseMemoUpdated` event: case_id, old_memo, new_memo
- [ ] #4 `CaseDeleted` event: case_id, name, member_count

### Membership Events
- [ ] #5 `CaseMemberAdded` event: case_id, source_id, link_type (FULL|PARTIAL), position (optional)
- [ ] #6 `CaseMemberRemoved` event: case_id, source_id

### Attribute Type Events
- [ ] #7 `AttributeTypeCreated` event: attribute_type_id, name, value_type (character|numeric|date)
- [ ] #8 `AttributeTypeRenamed` event: attribute_type_id, old_name, new_name
- [ ] #9 `AttributeTypeDeleted` event: attribute_type_id, name, affected_cases_count

### Attribute Value Events
- [ ] #10 `AttributeValueSet` event: case_id, attribute_type_id, old_value, new_value
- [ ] #11 `AttributeValueCleared` event: case_id, attribute_type_id, old_value

### Batch Events
- [ ] #12 `CasesImported` event: case_ids (list), source (csv|xlsx), row_count

### Event Structure
- [ ] #13 All events follow standard structure: event_id, event_type, aggregate_id, occurred_at, caused_by, payload
- [ ] #14 Events registered in Case context event catalog
<!-- AC:END -->
