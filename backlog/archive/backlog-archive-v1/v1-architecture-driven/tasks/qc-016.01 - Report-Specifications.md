## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Report type specifications: FrequencyReport, CodingSummaryReport, CoderComparisonReport, CoOccurrenceMatrix. Each with filter/grouping options.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
### Report Value Objects (frozen dataclasses)
- [ ] #1 `FrequencyReport`: code_frequencies (List[CodeFrequency]), filters, generated_at
- [ ] #2 `CodingSummaryReport`: code_summaries (List[CodeSummary]), filters, totals
- [ ] #3 `CoderComparisonReport`: agreements, disagreements, kappa_score
- [ ] #4 `CoOccurrenceMatrix`: matrix (Dict[CodeId, Dict[CodeId, int]]), codes

### Supporting Value Objects
- [ ] #5 `CodeFrequency`: code_id, code_name, count, percentage, sources
- [ ] #6 `CodeSummary`: code_id, segments (List[SegmentSummary]), total_characters
- [ ] #7 `CoOccurrenceCell`: code_a, code_b, overlap_count, edge_count

### Filter Specifications (frozen dataclasses)
- [ ] #8 `ReportFilters`: code_ids, source_ids, case_ids, coder_ids, date_range
- [ ] #9 All filter fields are Optional - None means "all"
- [ ] #10 Filters are composable and immutable

### Pure Functions
- [ ] #11 `apply_filters(segments, filters) -> List[Segment]` - pure filtering
- [ ] #12 `calculate_percentage(count, total) -> float` - pure calculation

### Immutability
- [ ] #13 All report types are frozen dataclasses
- [ ] #14 Reports are generated, never modified
<!-- AC:END -->
