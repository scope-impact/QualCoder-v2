## Description

Define Qt-friendly payload dataclasses for the Case context. These are emitted by SignalBridge signals to UI widgets.

## Acceptance Criteria

### Case Payloads
- [ ] #1 `CaseCreatedPayload`: case_id (str), name (str), owner_name (str)
- [ ] #2 `CaseRenamedPayload`: case_id (str), old_name (str), new_name (str)
- [ ] #3 `CaseDeletedPayload`: case_id (str), name (str)
- [ ] #4 `CaseMemoUpdatedPayload`: case_id (str), memo (str?)

### Membership Payloads
- [ ] #5 `MemberAddedPayload`: case_id (str), case_name (str), source_id (str), source_name (str), position_start (int?), position_end (int?)
- [ ] #6 `MemberRemovedPayload`: case_id (str), source_id (str)

### Attribute Payloads
- [ ] #7 `AttributeTypeCreatedPayload`: attribute_type_id (str), name (str), value_type (str)
- [ ] #8 `AttributeTypeDeletedPayload`: attribute_type_id (str), name (str)
- [ ] #9 `AttributeValueSetPayload`: case_id (str), case_name (str), attribute_name (str), value (str), value_type (str)

### Batch Operation Payloads
- [ ] #10 `CasesImportedPayload`: file_path (str), cases_count (int)
- [ ] #11 `CasesExportedPayload`: file_path (str), cases_count (int), format (str)

### Design Rules
- [ ] #12 All payloads are frozen dataclasses
- [ ] #13 Use primitive types only
- [ ] #14 Include display-ready names alongside IDs
