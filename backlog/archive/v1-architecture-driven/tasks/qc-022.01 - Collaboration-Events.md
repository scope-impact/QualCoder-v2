Domain events: CoderRenamed, CoderDeleted, CodingOwnershipChanged, ConflictDetected, ConflictResolved, ComparisonCalculated, AICodePromoted.

## Acceptance Criteria
<!-- AC:BEGIN -->
### Coder Lifecycle Events (frozen dataclasses extending DomainEvent)
- [ ] #1 `CoderCreated` event: coder_id, name, coder_type (HUMAN|AI)
- [ ] #2 `CoderRenamed` event: coder_id, old_name, new_name
- [ ] #3 `CoderDeleted` event: coder_id, name, codings_reassigned_to

### Coding Ownership Events
- [ ] #4 `CodingOwnershipChanged` event: segment_id, old_coder_id, new_coder_id
- [ ] #5 `AICodePromoted` event: segment_ids (list), from_coder (AI), to_coder (human)

### Merge Events
- [ ] #6 `MergeStarted` event: merge_id, source_code_id, target_code_id
- [ ] #7 `ConflictDetected` event: merge_id, conflict_type, source_item, target_item
- [ ] #8 `ConflictResolved` event: merge_id, conflict_id, resolution_strategy
- [ ] #9 `MergeCompleted` event: merge_id, segments_reassigned

### Comparison Events
- [ ] #10 `ComparisonCalculated` event: coder_a, coder_b, kappa_score, agreement_count

### AI-Specific Events
- [ ] #11 `AICoderAutoCreated` event: coder_id, agent_session_id - when AI first codes
- [ ] #12 `AIVisibilityToggled` event: visible (bool), affected_segments_count

### Event Structure
- [ ] #13 All events follow standard structure: event_id, event_type, aggregate_id, occurred_at, caused_by, payload
- [ ] #14 Events registered in Collaboration context event catalog
<!-- AC:END -->
