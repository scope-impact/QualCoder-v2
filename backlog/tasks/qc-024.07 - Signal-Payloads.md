## Description

Define Qt-friendly payload dataclasses for the Collaboration context. These are emitted by SignalBridge signals to UI widgets.

## Acceptance Criteria

### Coder Payloads
- [ ] #1 `CoderCreatedPayload`: coder_id (str), name (str), coder_type (str), is_ai (bool)
- [ ] #2 `CoderSwitchedPayload`: old_coder_id (str?), new_coder_id (str), new_coder_name (str)
- [ ] #3 `CoderVisibilityChangedPayload`: coder_id (str), coder_name (str), visible (bool)

### Merge Payloads
- [ ] #4 `MergeStartedPayload`: merge_id (str), source_project (str), target_project (str)
- [ ] #5 `ConflictDetectedPayload`: merge_id (str), conflict_type (str), source_item (str), target_item (str), description (str)
- [ ] #6 `ConflictResolvedPayload`: merge_id (str), conflict_index (int), resolution (str)
- [ ] #7 `MergeCompletedPayload`: merge_id (str), items_merged (int), conflicts_resolved (int)

### Comparison Payloads
- [ ] #8 `ComparisonGeneratedPayload`: comparison_id (str), coder_a (str), coder_b (str), kappa_score (float), agreement_percent (float)

### Promotion Payloads
- [ ] #9 `CodingPromotedPayload`: segment_id (str), from_coder (str), to_coder (str), code_name (str)

### Design Rules
- [ ] #10 All payloads are frozen dataclasses
- [ ] #11 Use primitive types only
- [ ] #12 Include coder names for display
