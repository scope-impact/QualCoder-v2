## Description

Define Qt-friendly payload dataclasses for the Source context. These are emitted by SignalBridge signals to UI widgets.

## Acceptance Criteria

### Source Payloads
- [ ] #1 `SourceImportedPayload`: source_id (str), name (str), media_type (str), file_path (str), size_bytes (int)
- [ ] #2 `SourceRenamedPayload`: source_id (str), old_name (str), new_name (str)
- [ ] #3 `SourceDeletedPayload`: source_id (str), name (str)
- [ ] #4 `SourceMemoUpdatedPayload`: source_id (str), memo (str?)
- [ ] #5 `TextExtractedPayload`: source_id (str), char_count (int), preview (str)

### Speaker Payloads
- [ ] #6 `SpeakerDetectedPayload`: source_id (str), speakers (list[str]), pattern_used (str)
- [ ] #7 `SpeakerCreatedPayload`: speaker_id (str), name (str), source_id (str)

### Transcript Payloads
- [ ] #8 `TranscriptUpdatedPayload`: source_id (str), transcript_preview (str), word_count (int)

### Case Linking Payloads
- [ ] #9 `CaseLinkedPayload`: source_id (str), case_id (str), case_name (str), position_start (int?), position_end (int?)
- [ ] #10 `CaseUnlinkedPayload`: source_id (str), case_id (str)

### Batch Operation Payloads
- [ ] #11 `FolderImportedPayload`: folder_path (str), sources_count (int), sources (list[str])
- [ ] #12 `SurveyImportedPayload`: file_path (str), cases_count (int), sources_count (int)

### Design Rules
- [ ] #13 All payloads are frozen dataclasses
- [ ] #14 Use primitive types only (str, int, float, bool)
- [ ] #15 Include display-ready data (e.g., case_name alongside case_id)
